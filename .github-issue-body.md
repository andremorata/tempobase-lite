## Summary

Create a zero-cost deployment option (TempoBase Lite) by porting the .NET backend into Next.js API routes within the existing `frontend/` app. Deploy to **Vercel** (free tier) with **Neon** PostgreSQL (free tier). Uses **Prisma** ORM with Neon serverless adapter, **Auth.js v5** for authentication. The existing frontend components, hooks, and pages remain largely unchanged ÔÇö the API client becomes environment-aware to route requests internally.

**Total monthly cost: $0**

---

## Motivation

The current architecture deploys to Azure Container Apps with managed PostgreSQL, which costs ~$15ÔÇô80+/month depending on the environment. For personal use, side-project demos, or small teams, a free-tier deployment on Vercel + Neon eliminates hosting costs entirely while preserving full feature parity.

**Key decisions:**
- Same repo ÔÇö API routes live inside `frontend/src/app/api/`
- Full feature parity ÔÇö all 29+ endpoints ported
- Dual-mode frontend ÔÇö `NEXT_PUBLIC_BACKEND_MODE` env var switches between `dotnet` (Azure) and `nextjs` (Vercel)
- Independent schemas ÔÇö Prisma schema maintained separately from EF Core (same logical schema, separate tooling)
- No data sync ÔÇö Neon and Azure PostgreSQL are completely separate database instances

## Free Tier Limits & Considerations

| Service | Limits | Notes |
|---------|--------|-------|
| **Vercel Free** | 100 GB bandwidth, 10s function timeout, 1 GB function memory, 100k function invocations/day | Generous for personal/small-team use. Large report generation may need pagination. |

- Vercel Pro ($20/mo): 60s timeout, 300k invocations/day
